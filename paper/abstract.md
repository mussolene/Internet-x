# Abstract

Internet-X is an identity-first overlay transport architecture built around an explicit resolution chain from human-readable names to stable cryptographic identities, then to mutable locators, and finally to selected paths. The project does not claim that identifier/locator separation, mobility support, or authenticated flow establishment are new in isolation; those topics have extensive prior art in systems such as HIP, LISP, ILNP, QUIC migration, and MOBIKE. Instead, Internet-X's contribution is the deliberate composition of these ideas into a packet-inspectable reference package with identity-bound flow establishment, authenticated locator rebinding, and explicit crypto-agility semantics for post-quantum transition. This repository provides a coherent specification set, a runnable Python reference implementation with real classical authenticated and encrypted flows, a bounded invariant model, reproducible tests and demos, and an honest evidence pack that states what is real, what is simulated, and what remains future work. Under the stated assumptions of overlay deployability, mobility-aware identity binding, and inspectable control semantics, Internet-X represents a strong design point rather than a claim of universal optimality.
